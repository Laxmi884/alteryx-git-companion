## Component Inventory

New components and modifications required in this phase.

### Modified: `ChangesPanel.tsx`

**Addition: Business context Textarea** (inside the `!hasAnyCommits` conditional block, after the amber warning Card)

- Element: `<Textarea>` (existing shadcn component at `src/components/ui/textarea.tsx`)
- Rows: 3 (matching commit message Textarea)
- Resize: `resize-none` (matching commit message Textarea)
- Placeholder: `"Describe what these workflows do and who uses them (optional)"`
- Label: `<h3 className="text-sm font-medium">Business context</h3>` above the Textarea — matches "Changed files" label pattern
- Visibility: shown only when `!hasAnyCommits` — disappears after first save, never reappears
- Required: false — Save Version button enabled regardless
- State: `businessContext` string state, local to ChangesPanel, passed along with commit payload

### New: AI Summary Panel (inside `DiffViewer.tsx`)

**Placement:** Below the sticky header, above the diff iframe. Full-width strip, collapsible by default.

**States the panel cycles through:**

1. **Idle** (button visible, panel empty): Shows "Generate AI summary" Button (primary variant, text-sm). Shown for all diffs where `isFirstCommit === false`.
2. **Loading / streaming**: Button replaced by step-label area. Spinner (animate-spin, w-4 h-4, inline) + current step label text (text-sm, muted-foreground). Completed steps shown as plain text-xs muted list above.
3. **Result**: Full narrative text rendered in a Card (CardContent, text-sm, normal weight). Risks (if any) shown as a muted text-xs list below narrative. "Regenerate" button (outline variant, text-xs) appears at panel bottom-right.
4. **Unavailable (no extras)**: Single line `text-sm text-muted-foreground`: "AI summary unavailable — install LLM extras"
5. **Unavailable (no model)**: Single line `text-sm text-muted-foreground`: "No LLM model configured"
6. **Error**: Single line `text-sm text-muted-foreground`: "Summary failed — {reason}. Retry?" with inline text-primary Retry link.

**Structure:**
```
<div className="border-b px-4 py-3 bg-background shrink-0">
  {/* idle: Button */}
  {/* loading: inline spinner + step labels */}
  {/* result: Card with narrative + optional risks list */}
  {/* unavailable/error: single muted line */}
</div>
```

The panel is a fixed-height strip (`min-h-[44px]`) that expands to fit content. It does not scroll independently — the diff iframe below it adjusts to fill remaining height.

### New: `useAISummary` hook (`src/hooks/useAISummary.ts`)

Follows the `useWatchEvents.ts` EventSource pattern. Opens `EventSource` on trigger call, closes on `result` event or error. Returns `{ status, steps, narrative, risks, trigger, reset }`.

## Interaction Contract

### Business Context Field

- Appears in `ChangesPanel` only when `hasAnyCommits === false`
- Field is optional — Save Version button not gated on it
- On save: `businessContext` value included in POST `/api/save/commit` body; backend writes `.acd/context.json` if non-empty
- After first save: field is permanently hidden (condition: `hasAnyCommits` flips to true)
- No character limit enforced in UI (backend may truncate; frontend does not)

### AI Summary Trigger

- Shown in `DiffViewer` for all entries where `isFirstCommit === false`
- Hidden when `isFirstCommit === true` (first commit already shows existing "no previous version" message)
- User clicks "Generate AI summary" button → opens EventSource to `GET /api/ai/summary?folder=...&sha=...&file=...`
- EventSource is request-scoped: opened on button click, closed when `result` event arrives or on error
- While streaming: button disappears, replaced by step label + spinner
- On `result` event: EventSource closed, narrative rendered in Card, "Regenerate" button appears
- On error: EventSource closed, error message shown with Retry link
- Retry / Regenerate: resets state, re-opens EventSource

### SSE Progress Rendering

- Each `{"type": "progress", "step": "..."}` event appends the step string to a `steps: string[]` list
- The most recent step is shown with a spinner; prior steps shown as muted completed list
- `{"type": "result", "narrative": "...", "risks": [...]}` event triggers final render and closes EventSource

### Layout Behavior

- AI summary panel sits between sticky DiffViewer header and diff iframe
- Panel height is auto (shrinks when idle, expands during streaming/result)
- Diff iframe fills remaining flex height below the panel — `flex-1` on the iframe container
- On mobile/narrow: panel wraps gracefully; button is full-width at <640px

## Accessibility

- Business context Textarea: `id="business-context"`, `<label htmlFor="business-context">` — matches existing Checkbox/label pattern
- AI summary panel: `role="status"` + `aria-live="polite"` on the step-label area so screen readers announce progress
- "Generate AI summary" Button: disabled state while streaming (`disabled={isStreaming}`)
- Spinner: `aria-hidden="true"` + visually hidden `<span className="sr-only">Loading...</span>`
- Muted unavailability messages: not `aria-hidden` — they are informational
