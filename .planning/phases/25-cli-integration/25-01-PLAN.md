# Plan 1: Change Narrative Generation + HTMLRenderer doc_fragment Kwarg

## Goal
Add the `generate_change_narrative()` async function and its Pydantic output model, and wire an opt-in `doc_fragment` kwarg into `HTMLRenderer.render()` so a narrative HTML section can be embedded in diff reports.

## Context
This plan builds the **non-CLI infrastructure** for CLI-02. It has zero user-facing surface on its own — it only adds LLM-pipeline and renderer hooks that Plan 03 will call from the `diff --doc` flag. Splitting this out lets Plans 02 and 03 run in parallel once Plan 01 lands.

Depends on: nothing (all Phase 23/24 APIs consumed are already complete).

Relevant decisions: D-13 (change narrative only, not intent doc), D-14 (new function in `doc_graph.py`), D-15 (new `doc_fragment` kwarg following existing optional-kwarg pattern).

### Task 2: Add `generate_change_narrative()` to doc_graph.py
**File:** `src/alteryx_diff/llm/doc_graph.py`
**Action:** Add new async function (no LangGraph graph — simple single-shot structured call)

Append after `generate_documentation()`. Update `__all__` to include `"generate_change_narrative"`.

Signature:
```python
async def generate_change_narrative(
    context: dict,              # ContextBuilder.build_from_diff() output: {summary, changes}
    llm: BaseChatModel,
) -> "ChangeNarrative":
    """Generate a change narrative for a diff using a single structured LLM call.

    Unlike generate_documentation(), this does not use a LangGraph multi-node
    pipeline — the input context is already a compact summary+changes dict, so
    a single structured_output call is sufficient.

    Args:
        context: ContextBuilder.build_from_diff() output dict with {summary, changes}.
        llm: Any LangChain BaseChatModel.

    Returns:
        A validated ChangeNarrative instance.
    """
```

Implementation notes:
- Import `ChangeNarrative` at function top (module-level import is also fine since `llm/models.py` has no langchain deps): `from alteryx_diff.llm.models import ChangeNarrative`
- Build `SystemMessage` with grounding constraint: "Only describe changes present in the provided diff context. Do not invent tools, IDs, or behaviors. Be concrete about what was added, removed, or modified."
- Build `HumanMessage` that includes `json.dumps(context, indent=2)` and asks for (1) a 2-4 paragraph narrative explaining the functional impact and (2) a list of production/risk concerns (may be empty)
- Call `llm.with_structured_output(ChangeNarrative, method="json_schema")` and `await structured_llm.ainvoke([system_msg, human_msg])`
- Return the result directly. No retry loop — callers can rerun if needed. Single `await`, no LangGraph state machine.

**Verification:** `python -m pytest tests/llm/test_doc_graph.py -q` still passes (no regressions). New function importable: `python -c "from alteryx_diff.llm.doc_graph import generate_change_narrative; print(generate_change_narrative)"`.

### Task 3: Add `to_html_fragment_from_narrative()` to DocRenderer
**File:** `src/alteryx_diff/renderers/doc_renderer.py`
**Action:** Add new method on existing `DocRenderer` class

Add a new Jinja template constant near `_HTML_TEMPLATE` (call it `_NARRATIVE_HTML_TEMPLATE`) that renders a `<section class="change-narrative" id="change-narrative">` block containing:
- An `<h2>` heading "AI Change Narrative"
- A prose `<div class="narrative-body">` with the `narrative` text (autoescape via Jinja — prose is plain text, not HTML; wrap paragraphs by splitting on `\n\n` if present)
- A `<ul class="narrative-risks">` listing risks (omit entirely if `risks` is empty)

Add method signature:
```python
def to_html_fragment_from_narrative(self, narrative: "ChangeNarrative") -> str:
    """Render a ChangeNarrative to an HTML <section> fragment for embedding in the diff report.

    Returns:
        An HTML string starting with ``<section class="change-narrative" id="change-narrative">``.
        Safe to pass as ``doc_fragment`` to ``HTMLRenderer.render()``.
    """
```

Use the same Jinja `Environment(autoescape=True)` pattern already in the class (`self._env_html` or whatever is already there — match the existing `to_html_fragment` pattern exactly). Type annotation uses `TYPE_CHECKING` import for `ChangeNarrative` to keep `doc_renderer.py` free of langchain-at-runtime imports (it already imports from `alteryx_diff.llm.models` which is pure Pydantic — that is fine).

The `id="change-narrative"` is load-bearing: Plan 03 tests assert its presence/absence.

**Verification:** Unit test in `tests/test_doc_renderer.py` (add to existing file):
```python
def test_to_html_fragment_from_narrative_embeds_id_and_body():
    from alteryx_diff.llm.models import ChangeNarrative
    from alteryx_diff.renderers.doc_renderer import DocRenderer
    narrative = ChangeNarrative(narrative="Para one.\n\nPara two.", risks=["risk-a"])
    html = DocRenderer().to_html_fragment_from_narrative(narrative)
    assert 'id="change-narrative"' in html
    assert "Para one." in html
    assert "risk-a" in html

def test_to_html_fragment_from_narrative_omits_risks_when_empty():
    from alteryx_diff.llm.models import ChangeNarrative
    from alteryx_diff.renderers.doc_renderer import DocRenderer
    html = DocRenderer().to_html_fragment_from_narrative(ChangeNarrative(narrative="x"))
    assert "narrative-risks" not in html
```

### Task 4: Add `doc_fragment` kwarg to `HTMLRenderer.render()` + template injection
**File:** `src/alteryx_diff/renderers/html_renderer.py`
**Action:** Modify existing `render()` method and `_TEMPLATE` string

1. Add a new keyword-only parameter to `HTMLRenderer.render()` immediately after `metadata`:
   ```python
   doc_fragment: str = "",
   ```
   Update the docstring — one paragraph matching the existing `graph_html` description style: "Optional HTML fragment (typically from `DocRenderer.to_html_fragment_from_narrative`) embedded after the graph and before the governance metadata. When empty (default), the HTML report is unchanged — zero regression risk."

2. Inside `render()`, pass `doc_fragment=doc_fragment` to `template.render(...)` alongside `graph_html` and `metadata`.

3. Modify `_TEMPLATE`: insert `{{ doc_fragment | safe }}` on its own line immediately after `{{ graph_html | safe }}` (line 353 of the current file) and before `{% if metadata %}` (line 354). Do not add any surrounding markup — the fragment already wraps itself in `<section>`.
