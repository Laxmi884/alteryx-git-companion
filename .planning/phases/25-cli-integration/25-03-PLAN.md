# Plan 3: `diff --doc` Flag — Embed AI Change Narrative in HTML Report

## Goal
Add an opt-in `--doc` flag to the existing `diff` command that calls `generate_change_narrative()` and injects the rendered narrative fragment into the HTML report via the `doc_fragment` kwarg added in Plan 01.

## Context
This plan completes CLI-02. It consumes the infrastructure from Plan 01 (`generate_change_narrative`, `ChangeNarrative`, `DocRenderer.to_html_fragment_from_narrative`, `HTMLRenderer.render(..., doc_fragment=...)`) and reuses the `_resolve_model_string` / `_resolve_llm` helpers added to `cli.py` in Plan 02. Without those helpers, Plan 03 would duplicate the model resolution logic.

**Dependency order:** Plan 03 depends on Plan 01 AND Plan 02 (it imports the helpers added in Plan 02 and the new function from Plan 01). It is the third wave.

Relevant decisions: D-12 (opt-in, absent by default), D-13 (narrative only, uses `build_from_diff`), D-14 (calls `generate_change_narrative`), D-15 (passes result via `doc_fragment`), D-16 (same spinner), D-17 (`asyncio.run()`).

Relevant pitfalls (from research): Pitfall 5 — `result.is_empty` early exit currently happens at line 125 of `cli.py`; the LLM call must be gated AFTER that check so `--doc` on an identical pair does nothing (no LLM spend, no narrative section). Pitfall 3 — `asyncio.run()` nested loop is out-of-scope for Phase 25 (CLI only).

### Task 1: Add `doc` option to `diff` command and wire narrative pipeline
**File:** `src/alteryx_diff/cli.py`
**Action:** Modify existing `diff()` function — add one new option, add LLM call block, pass result to `HTMLRenderer.render()`

**1a — New option** (append to existing `diff()` signature, after `json_output`):
```python
doc: bool = typer.Option(  # noqa: B008
    False,
    "--doc",
    help="Embed an AI-generated change narrative section in the HTML report (requires [llm] extras and a configured model).",
),
model: str | None = typer.Option(  # noqa: B008
    None,
    "--model",
    help="LLM provider and model, e.g. ollama:llama3 (used only when --doc is set). Falls back to ACD_LLM_MODEL env var, then config_store.",
),
base_url: str | None = typer.Option(  # noqa: B008
    None,
    "--base-url",
    help="Override LLM endpoint URL (used only when --doc is set).",
),
```

**1b — Narrative generation block.** Insert after the `if result.is_empty:` early-exit block and before the HTML render branch. Guard the entire block with `if doc and not json_output:`. When `doc=True` and `json_output=True`, emit a stderr warning "Note: --doc has no effect with --json; narrative skipped." and do not call the LLM.

Behavior:
```python
doc_fragment = ""
if doc and not json_output:
    # D-10: require LLM extras
    try:
        from alteryx_diff.llm import require_llm_deps
        require_llm_deps()
    except ImportError as e:
        typer.echo(f"Error: {e}", err=True)
        raise typer.Exit(code=2) from None

    # Reuse helpers from Plan 02
    model_str = _resolve_model_string(model)
    llm = _resolve_llm(model_str, base_url)

    # Deferred imports — CORE-01 compliance
    from alteryx_diff.llm.context_builder import ContextBuilder
    from alteryx_diff.llm.doc_graph import generate_change_narrative
    from alteryx_diff.renderers.doc_renderer import DocRenderer
    import asyncio

    narrative_context = ContextBuilder.build_from_diff(result)

    if quiet:
        narrative = asyncio.run(generate_change_narrative(narrative_context, llm))
    else:
        with _err_console.status("Generating change narrative...", spinner="dots"):
            narrative = asyncio.run(generate_change_narrative(narrative_context, llm))

    doc_fragment = DocRenderer().to_html_fragment_from_narrative(narrative)
elif doc and json_output:
    if not quiet:
        typer.echo("Note: --doc has no effect with --json; narrative skipped.", err=True)
```

**1c — Pass to renderer.** Update the existing `HTMLRenderer().render(...)` call to add `doc_fragment=doc_fragment` as a new kwarg (will be `""` when `--doc` not passed → zero regression on the no-doc path).

**Verification:** Syntax check passes; `python -c "from alteryx_diff.cli import diff; print(diff)"` works. The existing `tests/test_cli.py` (now expecting `"diff"` prefix after Plan 02) still passes without any modifications in this plan — confirming opt-in non-regression.
